เทคนิคและ Best Practices ในการจัดการ Dependencies & เขียนโค้ด

1. เวอร์ชันล็อกให้ชัดเจน
   กำหนดเวอร์ชันที่เหมาะสม (เช่น 3.3.2 แทน latest) เพื่อลดปัญหาจากการอัปเดตอัตโนมัติที่อาจทำให้โค้ดไม่ทำงาน

2. แยก dependencies กับ devDependencies ให้ชัดเจน

dependencies สำหรับโค้ดที่รันจริงบนโปรดักชัน

devDependencies สำหรับเครื่องมือพัฒนา เช่น linter, build tools

3. อัปเดต dependency อย่างสม่ำเสมอ
   แต่ให้ทดสอบก่อนว่าการอัปเดตเวอร์ชันใหม่ไม่ทำให้ระบบเสียหาย

4. ใช้เครื่องมือจัดการเวอร์ชัน
   เช่น pnpm ที่เร็วและประหยัดพื้นที่กว่า npm/yarn

5. ตั้งค่า ESLint + Prettier ร่วมกัน
   เพื่อความสม่ำเสมอและลดข้อผิดพลาดเชิงไวยากรณ์ในโค้ด

6. ใช้ TailwindCSS plugin แยกให้ถูกต้อง
   Tailwind 4.x แยก plugin postcss ออกมา ต้องเพิ่ม @tailwindcss/postcss เพื่อไม่ให้เกิด error

7. ใช้ alias path ใน TypeScript และ Vite
   ลดการเขียน path ยาวๆ ช่วยให้โค้ดดูสะอาดและง่ายต่อการ refactor

8. ตั้งค่า husky + lint-staged
   เพื่อรัน linter ก่อน commit ป้องกันโค้ดไม่ดีเข้าสู่ repo

9. ใช้ Vite เพื่อ build
   เน้นความเร็วในการพัฒนาและ build

10. แยก theme และ styling ออกจาก logic
    เช่นใช้ daisyUI + TailwindCSS ช่วยแยกหน้าที่ชัดเจนและง่ายต่อการปรับธีม
